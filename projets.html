Deploiement d'architecture reseau d'entreprise

Dec. 2025 - Janv. 2026 | Individuel | 17.5 h | S1

Contexte

Ce projet sert a comprendre les bases d'une infrastructure d'entreprise avant le passage sur du materiel reel : segmentation, adressage, services et routage.

Realisations techniques

J'ai construit une topologie virtualisee, defini un plan d'adressage coherent, separe le trafic par VLAN, configure VTP, les serveurs DHCP et le routage inter-VLAN. Le travail inclut aussi des tests de connectivite et du diagnostic lorsque certains flux ne passaient pas.

Captures du projet

Technologies

EVE-NGCiscoVLANVTPDHCPRoutage inter-VLAN

Competences acquises

  • AC11.02 - Comprendre l'architecture et les fondements des systemes numeriques, des communications et de l'Internet.
  • AC11.03 - Configurer les fonctions de base du reseau local.
  • AC11.04 - Maitriser les roles des systemes d'exploitation pour administrer reseaux et services.

Bilan reflexif personnel

Ce projet m'a montre qu'une bonne preparation evite beaucoup d'erreurs. Je retiens surtout l'importance du plan d'adressage et de la documentation, car une petite incoherence sur un VLAN peut bloquer toute une partie du reseau.

SAE Intégrative 2.04

SAE 2.04 | Projet intégratif | Web & BDD | S2

Contexte global du projet

La SAE Intégrative 2.04 regroupait plusieurs parties : réseau, téléphonie, collecte de données, web et base de données. Le but était de créer une infrastructure complète capable de récupérer des données de capteurs, de les stocker et de les afficher dans une application web.

Les données provenaient de capteurs de température et permettaient de surveiller plusieurs pièces comme la chambre ou le séjour. L’application permettait aussi de consulter les mesures, comparer les espaces surveillés et gérer les capteurs.

Mon travail personnel

Personnellement, je me suis occupé de la partie web et de la base de données. J’ai travaillé sur l’interface Django, l’affichage des mesures, les graphiques, les filtres, l’export CSV et la gestion des capteurs.

Pour la base de données, j’ai réalisé la partie BDD avec MySQL afin d’organiser les capteurs et les mesures de température de manière claire et exploitable par le site.

Fonctionnalités réalisées

  • Affichage des dernières mesures sous forme de tableau.
  • Affichage graphique des températures par pièce.
  • Filtres par capteur et par période.
  • Comparaison des espaces surveillés.
  • Export des données au format CSV.
  • Gestion des capteurs avec modification et suppression.
  • Mise à jour automatique de l’interface.

Captures du projet

Technologies

DjangoHTMLCSSJavaScriptMySQL

Compétences acquises

  • AC13.02 - Lire, modifier et corriger un programme.
  • AC13.04 - Connaître l’architecture et les technologies d’un site web.
  • AC13.05 - Choisir les mécanismes de gestion de données adaptés.
  • AC13.06 - S’intégrer dans un environnement de développement.

English summary

This project was about creating a complete system to monitor temperature sensors. The global project included networking, data collection, a web application and a database. My personal work focused on the web interface and the MySQL database. I used Django, HTML, CSS, JavaScript and MySQL to display, filter and manage sensor data.

Bilan réflexif personnel

Ce projet m’a permis de mieux comprendre comment plusieurs parties techniques peuvent fonctionner ensemble : réseau, collecte de données, base de données et interface web. J’ai surtout progressé sur le lien entre Django et MySQL, ainsi que sur l’affichage et la gestion des données dans une application web.
